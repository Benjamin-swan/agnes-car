# PLAN.md — 작업 기획 및 진행 현황

## 목표
PRD(`agnes_car_management_PRD.md`) 기반, 아그네스 1대 전용 차량 관리 웹서비스를 프론트엔드로 완성하고 GitHub Pages에 배포한다.

## 확정 결정 사항
- 디자인: **클린 미니멀 라이트 / 딥그린(#1B6B4C)** (→ DESIGN.md)
- 저장: **localStorage + JSON 백업**(내보내기/가져오기)
- 사진 첨부: **v1 제외** (스키마는 확장 여지 남김)
- 알림: 정적 사이트 특성상 실시간 푸시 불가 → **대시보드 시각 경고**로 대체
- 배포: 사용자 GitHub 계정으로 **GitHub Pages** 직접 배포

## 화면(PRD 4장)
1. **홈(대시보드)** — 차량 정보, 소모품 상태바, 최근 활동 타임라인, 이달 지출 요약
2. **정비/활동 등록** — 카테고리(정비/주유/세차/부품)별 가변 폼
3. **통합 이력 조회** — 타임라인 + 유형 필터
4. **알림/환경 설정** — 소모품 교체 주기 편집, 차량 프로필, 백업 import/export

## 기능 매핑(PRD 3장 → 구현)
- 3.1 차량 프로필 관리 → 설정 화면 프로필 편집 + 현재 주행거리 갱신
- 3.2 정비 이력 → 등록/이력 (사진 첨부는 v1 제외)
- 3.3 주유/세차/부품 → 등록 가변 폼 + 리터당 단가·연비 자동계산
- 3.4 교체 주기·예측 → 프리셋 + 커스텀 편집 + **일평균 주행거리 추정** → 예측 주행거리 → 상태 판정
- 3.5 대시보드/통계 → 상태바, 월별/항목별 지출 막대차트, 연비 추이 라인차트
- 5장 예외처리 → 주행거리 역전 경고, 부품구매→정비 이력 전환 버튼

## 예측 로직(핵심)
- 일평균 주행거리 = (최신 주행거리 − 과거 기준 주행거리) / 경과 일수. 기록 2건 이상이면 최근 구간으로 산정, 없으면 기본값(예: 40km/일) 사용.
- 예측 현재 주행거리 = 최신 입력 주행거리 + 일평균 × (오늘 − 최신 입력일)
- 소모품 잔여 = min(주기km − (예측주행 − 마지막교체주행), 주기월 환산 잔여). 잔여 비율로 정상/주의/교체 판정(주의 ≤ 15%, 교체 ≤ 0%).
